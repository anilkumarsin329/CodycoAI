# Global Language Translator Implementation - Anil.txt

## Overview
Maine React mein global language translator banaya hai jo puri app mein work karta hai. Yeh Context API use karta hai.

## 1. Context API Setup (LanguageContext.jsx)

```javascript
import { createContext, useContext, useState } from 'react';

// Context banaya
const LanguageContext = createContext();

// Custom hook banaya context use karne ke liye
export const useLanguage = () => {
  const context = useContext(LanguageContext);
  if (!context) {
    throw new Error('useLanguage must be used within a LanguageProvider');
  }
  return context;
};

// Provider component banaya
export const LanguageProvider = ({ children }) => {
  const [language, setLanguage] = useState('EN'); // Default language

  // Translations object - sab languages ka data
  const translations = {
    EN: {
      home: "Home",
      features: "Features",
      about: "About us",
      // ... more translations
    },
    HI: {
      home: "होम",
      features: "विशेषताएँ", 
      about: "हमारे बारे में",
      // ... more translations
    }
  };

  // Translation function - key pass karo, translation milega
  const t = (key) => {
    return translations[language][key] || key;
  };

  // Provider return karta hai value
  return (
    <LanguageContext.Provider value={{ language, setLanguage, t }}>
      {children}
    </LanguageContext.Provider>
  );
};
```

## 2. App.jsx mein Provider Wrap kiya

```javascript
import { LanguageProvider } from './contexts/LanguageContext'

function App() {
  return (
    <LanguageProvider>  {/* Puri app ko wrap kiya */}
      <div>
        <Navbar />
        <Home/>
        <TheProblem/>
        <Footer />
      </div>
    </LanguageProvider>
  )
}
```

## 3. Components mein Use kiya

### Navbar.jsx Example:
```javascript
import { useLanguage } from "../../contexts/LanguageContext";

export default function Navbar() {
  const { language, setLanguage, t } = useLanguage(); // Hook use kiya
  
  return (
    <nav>
      {/* Language dropdown */}
      <button onClick={() => setLanguage(language === 'EN' ? 'HI' : 'EN')}>
        {language}
      </button>
      
      {/* Translated text */}
      <a href="#features">{t('features')}</a>
      <a href="#about">{t('about')}</a>
    </nav>
  );
}
```

### TheProblem.jsx Example:
```javascript
import { useLanguage } from "../../contexts/LanguageContext";

export default function TheProblem() {
  const { t } = useLanguage(); // Sirf t function chahiye
  
  return (
    <section>
      <p>{t('theProblem')}</p>  {/* "The problem" ya "समस्या" */}
      <h2>{t('problemTitle')}</h2>
      <h3>{t('staffLack')}</h3>
      <p>{t('staffLackDesc')}</p>
    </section>
  );
}
```

## 4. How It Works (Kaise Kaam Karta Hai)

### Step 1: Context Creation
- `createContext()` se global context banaya
- Yeh data store karta hai jo sab components access kar sakte hain

### Step 2: Provider Setup  
- `LanguageProvider` component banaya jo:
  - Current language state maintain karta hai
  - Translation function provide karta hai
  - Language change function provide karta hai

### Step 3: App Wrapping
- Puri app ko `LanguageProvider` se wrap kiya
- Ab sab child components context access kar sakte hain

### Step 4: Component Usage
- Har component mein `useLanguage()` hook use kiya
- `t('key')` function se translation get kiya
- Language change karne se sab components re-render hote hain

## 5. Key Benefits

1. **Global State**: Ek jagah language change karo, sab jagah update ho jaye
2. **Easy to Use**: Sirf `t('key')` likhna hai
3. **Scalable**: Naye languages easily add kar sakte hain
4. **Type Safe**: Keys missing hain to fallback milta hai
5. **Performance**: Sirf language change pe re-render hota hai

## 7. Usage Pattern

```javascript
// Component mein import karo

// Hook use karo

// Text translate karo
```
